Data Types & Column Constraints

Common SQL data types and constraints (from your PDF): INT / DEC / CHAR / VARCHAR / DATE / TIME, and constraints like PRIMARY KEY, AUTO_INCREMENT, NOT NULL, UNIQUE, DEFAULT.

Example: create a table with constraints

Copied! 
CREATE TABLE students (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  dob DATE,
  gpa DECIMAL(4,2) DEFAULT 0.00
);

Example: CHAR vs VARCHAR explanation

Copied! 
-- CHAR(10) reserves fixed 10 bytes; VARCHAR(10) uses needed bytes
CREATE TABLE sample (
  fixed CHAR(10),
  variable VARCHAR(50)
);

DDL — Data Definition Language

Commands for creating/modifying/dropping tables: CREATE TABLE, ALTER TABLE, DROP TABLE.

CREATE TABLE example

Copied! 
CREATE TABLE employee (
  Emp_code INT PRIMARY KEY,
  Emp_name VARCHAR(50),
  Designation VARCHAR(50),
  Department VARCHAR(50),
  Basic DECIMAL(10,2)
);

ALTER TABLE (add / modify / drop column)

Copied! 
-- add a column
ALTER TABLE employee ADD COLUMN DA DECIMAL(10,2);

-- modify column type (MySQL syntax)
ALTER TABLE employee MODIFY COLUMN Emp_name VARCHAR(100);

-- drop a column
ALTER TABLE employee DROP COLUMN Designation;

DROP TABLE (delete table)

Copied! 
DROP TABLE IF EXISTS employee;

DML — Data Manipulation Language

Commands to insert, select, update and delete rows: INSERT INTO, SELECT, UPDATE, DELETE.

INSERT INTO (single & multiple rows)

Copied! 
INSERT INTO employee (Emp_code, Emp_name, Designation, Department, Basic) VALUES
(100, 'Arjun', 'Trainee', 'Sales', 16000.00),
(101, 'Maya', 'Trainee', 'Purchase', 9000.00);

SELECT examples

Copied! 
-- select everything
SELECT * FROM employee;

-- select with condition
SELECT Emp_name, Department FROM employee WHERE Department = 'Sales';

-- select with order
SELECT * FROM student ORDER BY batch, stud_name;

UPDATE example

Copied! 
-- compute DA = 75% of Basic (example from lab)
UPDATE employee SET DA = Basic * 75/100;

-- compute Gross_pay
UPDATE employee SET Gross_pay = Basic + DA;

DELETE example

Copied! 
-- delete a single row
DELETE FROM employee WHERE Emp_code = 999;

-- delete all rows (be careful)
DELETE FROM employee;

DCL — Data Control Language

Commands: GRANT, REVOKE.

GRANT example

Copied! 
-- grant SELECT on a table to user 'examinee' (MySQL)
GRANT SELECT ON mydb.student TO 'examinee'@'localhost';

REVOKE example

Copied! 
REVOKE SELECT ON mydb.student FROM 'examinee'@'localhost';

Aggregate Functions

SUM(), AVG(), MIN(), MAX(), COUNT() — useful for reports and analysis (explicit examples below).

Examples

Copied! 
-- total of all marks
SELECT SUM(total) AS GrandTotal FROM student;

-- average of mark1
SELECT AVG(mark1) AS AvgMark1 FROM student;

-- highest and lowest
SELECT MAX(unit_price) AS HighestPrice, MIN(qty) AS MinQty FROM stock;

-- count non-null rows
SELECT COUNT(*) AS TotalBooks FROM book;

VIEW — virtual tables

A view is a virtual table derived from one or more tables. Create with CREATE VIEW.

Copied! 
CREATE VIEW commerce_students AS
SELECT roll_number, stud_name, total
FROM student
WHERE batch = 'Commerce';

-- use it
SELECT * FROM commerce_students;

Practical Example Sets (ready to copy)

Grouped lab examples: Student / Employee / Stock / Bank / Book (taken & adapted from your earlier file content).

Student table — create, insert, update total, select

Copied! 
CREATE TABLE student (
  roll_number INT NOT NULL PRIMARY KEY,
  stud_name VARCHAR(25),
  batch VARCHAR(25),
  mark1 INT, mark2 INT, mark3 INT, total INT
);

INSERT INTO student VALUES 
(1, 'Alan Turing', 'Science', 80, 70, 55, NULL),
(2, 'Ada Lovelace', 'Science', 30, 40, 15, NULL),
(3, 'Adam Smith', 'Commerce', 70, 60, 40, NULL);

-- update totals
UPDATE student SET total = mark1 + mark2 + mark3;

-- list commerce students
SELECT * FROM student WHERE batch = 'Commerce';

Employee table — create, insert, update DA, gross

Copied! 
CREATE TABLE employee (
  Emp_code INT PRIMARY KEY,
  Emp_name VARCHAR(20),
  Designation VARCHAR(25),
  Department VARCHAR(25),
  Basic DECIMAL(10,2), DA DECIMAL(10,2), Gross_pay DECIMAL(10,2)
);

INSERT INTO employee VALUES 
(100, 'Arjun', 'Trainee', 'Sales', 16000.00, NULL, NULL),
(101, 'Maya', 'Trainee', 'Purchase', 9000.00, NULL, NULL);

-- compute DA = 75% of Basic
UPDATE employee SET DA = Basic * 75/100;

-- compute gross pay
UPDATE employee SET Gross_pay = Basic + DA;

-- employees with gross pay < 20000
SELECT * FROM employee WHERE Gross_pay < 20000;

Stock table — zero stock, counts, price update

Copied! 
CREATE TABLE stock (
  Item_code INTEGER PRIMARY KEY,
  Item_name VARCHAR(20),
  Manufacturer_Code VARCHAR(10),
  Qty INT, Unit_Price DECIMAL(10,2)
);

INSERT INTO stock VALUES 
(1001, 'Smart Phone', 'Samsung', 23, 36000),
(1005, 'Toaster', 'Philips', 0, 3000),
(1007, 'Coffee Maker', 'Havels', 0, 16000);

-- find zero stock
SELECT * FROM stock WHERE Qty = 0;

-- count by manufacturer
SELECT Manufacturer_Code, COUNT(Item_code) AS No_Products FROM stock GROUP BY Manufacturer_Code;

-- increase price by 10%
UPDATE stock SET Unit_Price = Unit_Price * 1.10;

Book table — select, SCERT update, like pattern

Copied! 
CREATE TABLE book (
  Book_ID INT NOT NULL PRIMARY KEY,
  Book_Name VARCHAR(50), Author_Name VARCHAR(50),
  Pub_Name VARCHAR(50), Price DECIMAL(10,2)
);

INSERT INTO book VALUES
(1200, 'The Kite Runner', 'Khaled Hosseini', 'River Head', 450),
(1207, 'Art of Computer Programming', 'Knuth', 'SCERT', 125),
(1209, 'Python Programming', 'Michael Dawson', 'Pearson', 3300);

-- SCERT books
SELECT * FROM book WHERE Pub_Name = 'SCERT';

-- increase SCERT price 10%
UPDATE book SET Price = Price * 1.10 WHERE Pub_Name = 'SCERT';

-- names ending with 'Programming'
SELECT * FROM book WHERE Book_Name LIKE '%Programming';

How to Use / Tips

  • Click any Copy button to copy that code block to clipboard and paste into your SQL client (MySQL / MariaDB / SQLite / Postgres with small tweaks).
  • Most numeric strings in the examples are plain numbers; remove quotes if your SQL engine enforces numeric types (e.g., INSERT INTO stock uses numbers without quotes).
  • ALTER / DROP are irreversible on live data — use CREATE TABLE IF NOT EXISTS and DROP TABLE IF EXISTS during testing.
  • Aggregate functions (SUM/AVG) operate on numeric columns — ensure totals are updated first (e.g., UPDATE student SET total = mark1+mark2+mark3).
